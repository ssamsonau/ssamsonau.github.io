3D plots as in Mathematica

Sometimes you just need to visualize a shape of a function. Ideally this should be fast, intuitive and simple. I always liked the simplicity of Wolfram Mathematica 3D plots. Just do

Plot3D[Sin[x*y], {x, 0, 5}, {y, 0, 5}]

and you have a very nice plot. Which is, by the way, interactive (as long as you stay in Mathematica)

Overview of 2D and 3D

2D

In a case of two dimentional plots, functions curve and plot will do the job.

par(mfrow = c(1, 2))
curve(sin, -pi, pi*3) #plot will work as well

temp_fun <- function(x) sin(x) * x
curve(temp_fun, -pi, pi*3)


par(mfrow = c(1, 1))

3D

There are various packages which can do 3D plots in R like scatterplot3d, scatter3d, rgl, plot3D; in Python this can be done using matplotlib

And also there is plotly - a platform with an interface for both R and Python

Only rgl package provides a nice one-line interface out of the box. In order to use functions in other packages one have to generate values for x, y, and z. This is not hard, but does not look as neat as in Mathematica. Bellow are examples for

  • rgl in R (interactive both locally and in html created by knitr)
  • plotly in R (interactive both locally and in html created by knitr)
  • Python matplotlib (interactive only locally)

Simplest way with rgl package

rgl package is concise already (one can use a function Vectorize, which will turn any function into a it’s vectorized analog).

In order to include plot produced by rgl packages into html created by knitr, one would need to specify 1) knitr::knit_hooks$set(webgl = rgl::hook_webgl) and 2) option webgl=TRUE in a chunk with code producing a plot. More details here

library(rgl)
ffv <- Vectorize(function(x, y) sin(x*y))

persp3d(ffv, xlim = c(0, 5), ylim = c(0, 5), col = plot3D::jet.col)

Plotly with R

Here is an example of a function one can do to reproduce Mathematica’s ‘one line behavior’ for plotly and plot3D

Function definition

#' Draw 3D surface plot of the specified function, by specifying only ranges of x and y
#' 
#' @param f a function of \code{x} and \code{y}
#' @param x_range a range of values for \code{x}: c(0, 1)
#' @param y_range a range of values for \code{y}: c(0, 1)
#' @param resolution Number of dots calcualted will be resolution^2
#' @return plot
#' @examples
#' surf_plot_function_of_xy(function(x, y) sin(x + y), c(0, 5), c(0, 5), resolution = 100)

surf_plot_function_of_xy <-
  function(f, x_range, y_range, resolution = 101) {
    
    x <- seq(from = x_range[1], to = x_range[2], length.out = resolution)
    y <- seq(from = y_range[1], to = y_range[2], length.out = resolution)
    
    grid_df <- expand.grid(x, y)
    
    library(dplyr)
    z <- mapply(f, grid_df$Var1, grid_df$Var2) %>%
      matrix(ncol = resolution)
    
    library(plotly)
    plot_ly(x = x, y = y, z = z) %>%
      add_surface()
  }

Plotly example

#Sys.setenv("plotly_username"="your_plotly_username")
#Sys.setenv("plotly_api_key"="your_api_key")
p <- surf_plot_function_of_xy(function(x, y) sin(x * y), c(0, 5), c(0, 5), resolution = 101)
api_create(p, filename = "plotly_3dSurface") # upload to plotly cloud

Python matplotlib

  • The code bellow based on example from this link

  • Python code can be executed in Rstudio and plot be included in the html file with reticulate package - more details here

    • If there is an error “could not find or load the Qt platform plugin windows”, look here for a solution - for example you may need to use a command Sys.setenv(QT_PLUGIN_PATH = “c:/Users/..your_user../Anaconda3/Library/plugins/")
from mpl_toolkits.mplot3d import Axes3D
import matplotlib.pyplot as plt
from matplotlib import cm
import numpy as np
fig = plt.figure()
ax = fig.gca(projection='3d')
# Make data.
X = np.arange(0, 5, 0.05)
Y = np.arange(0, 5, 0.05)
X, Y = np.meshgrid(X, Y)
Z = np.sin(X*Y)
# Plot the surface.
surf = ax.plot_surface(X, Y, Z, cmap=cm.coolwarm,
                       linewidth=0, antialiased=False)
# Add a color bar which maps values to colors.
fig.colorbar(surf, shrink=0.5, aspect=5)
plt.show()

This plot will be interactive while one uses Matplotlib’s interactive backend. The interactivity does not translate into Rmd generated html.

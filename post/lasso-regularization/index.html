Lasso regularization. An example helping to develop an intuition of why and how it works.

Cost function

In a typical case of regression a cost function (which is to be minimized) for lasso is defined like follows (where n is a number of observations, m - number of predictors)

$$ CostFunciton = RSS + penalty $$ $$ penalty = \lambda \sum_{i=i}^{m} |\beta_i|$$

$$ RSS = \frac{1}{n} \sum_{i=1}^{n} (y_i-\hat{y_i})^2 $$

$$ \hat{y_i} = \beta0 + \sum{j = 1}^{m} \beta_j * x_j$$

Helping functions

Let us define functions needed to calculate and plot both RSS and cost function.

library(pracma)
library(plotly)
library(tidyverse)
cost <- function(fit, df, name1, name2, beta1, beta2){
  fit$coefficients[[name1]] <- beta1
  fit$coefficients[[name2]] <- beta2
  y_hat <- predict(fit, df)
  RSS <- mean((df$y - y_hat)^2)
}

cost_vect <- Vectorize(cost, vectorize.args = c("beta1", "beta2"))

cost_xyz <- function(fit, df, name1, name2, resolution = 50){

  x_min <- fit$coefficients[[name1]]
  y_min <- fit$coefficients[[name2]]
  
  cat("minimum of RSS at", "\n")
  cat(name1, " ", x_min, "\n")
  cat(name2, " ", y_min, "\n")
  
  library(pracma)
  xy <- meshgrid(seq(0, fit$coefficients[[name1]]*1.5, length.out = resolution),
                 seq(0, fit$coefficients[[name2]]*1.5, length.out = resolution))
  
  z <- cost_vect(fit, df, name1, name2, xy$X, xy$Y) %>%
    matrix(ncol = ncol(xy$X))
  
  z_min <- cost_vect(fit, df, name1, name2, x_min, y_min)
   
  penalty <- abs(xy$X) + abs(xy$Y) # lasso
  penalty_ridge <- (xy$X)^2 + (xy$Y)^2
  
  list(xy = xy, z = z, penalty = penalty, penalty_ridge = penalty_ridge,
       min_xyz = c(x_min, y_min, z_min))
  
}

Thus such predictor is not very useful. And removing it should decrease variance - for a new data this ‘optimal beta for training set’ (which, as we see does not represent catching of an actual pattern), will be far from optimal. This will increase variance.

Parameter $\lambda$ will be chosen using cross validation (cv) or bootstrap. So let us assume for a moment that $\lambda$ is fixed. For a given run of cv with a fixed $\lambda$ parameters $\beta_1$ and $\beta_2$ will be chosen to minimize the cost function.

  • both RSS and penalty depend on $\beta_1$ and $\beta_2$
  • both RSS and penalty have to agree on

What can we say about relations of $\beta_1$ and $\beta_2$?

  • We know that equal values of penalty $\lambda \sum_{i=i}^{m} |\beta_i|$ are following lines displayed on the plot above
  • Thus for any combination of $\beta_1$ and $\beta2$, program can “move” along such line without changing the the part of cost coming from penalty $\lambda \sum{i=i}^{m} |\beta_i|$
  • Now looking at plot of RSS, we can imagine several lines crossing a chosen constant line of penalty. We need to choose one of these intersections - any of those will have the same penalty, but will correspond to different RSS.
  • One can see that in order to minimize RSS, one of $\beta$ will go to zero
p <- plot_ly(x = obj$xy$X[1, ], y = obj$xy$Y[, 1]) %>%

Conclusion, with a note about ridge

Thus we see that given two coefficients one of them will be driven to zero.
Which one? The one which changes the cost less than another - in this case first ‘x’, then ‘sin’, until only ‘cos’ is left.

In case of ridge regression coefficients are not driven to complete zero, due to the fact that penalty has a form $$ penalty = \lambda \sum_{i=i}^{m} \beta_i^2$$

Thus such penalty looks like a circle and does force one of coefficients to become zero: see the plot bellow

p <- plot_ly(x = obj$xy$X[1, ], y = obj$xy$Y[, 1]) %>%
  add_contour(z = obj$penalty_ridge, name = "penalty") %>%
  add_contour(z = obj$z, opacity = 0.5, name = "RSS") %>%
  add_trace(x = obj$min_xyz[1], y = obj$min_xyz[2],
